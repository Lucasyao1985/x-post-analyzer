# 抓取策略详解（v2 — 优化版）

## 回退优先级

### 1. fxtwitter API（优先级：最高，推荐）
- URL: `https://api.fxtwitter.com/{username}/status/{tweetId}`
- 工具: WebFetch
- 优点: 一次调用返回全部数据（文本+媒体+互动+用户信息），JSON格式，成功率高
- 缺点: 偶尔延迟或不可用
- **工具调用次数: 1**

### 2. 搜索引擎（优先级：中）
- 搜索: `site:x.com {username} status {tweetId}`
- 工具: WebSearch
- 优点: 可能找到缓存或转载内容
- 缺点: 可能无完整原文，无互动数据
- **工具调用次数: 1**

### 3. 直接访问 x.com（优先级：低，最后手段）
- URL: x.com, twitter.com
- 工具: WebFetch
- 优点: 最直接
- 缺点: 被反爬虫拦截概率极高
- **工具调用次数: 1**

## 工具调用预算

| 策略 | Fetch 阶段 | Analyze 阶段 | 总计 |
|------|-----------|-------------|------|
| v1（旧） | 3 Agent × 多次 = 10+ | 1 Agent × 多次 | **12-15 次** |
| v2（新） | 1 Agent × 1-3 次 | 0 次（纯文本） | **1-3 次** |

## 成功判断标准

| 数据完整性 | 判定 |
|-----------|------|
| 原文 + 互动数据 | ✓ 完全成功 |
| 原文，无互动数据 | ✓ 基本成功 |
| 部分内容 | △ 部分成功 |
| 仅搜索上下文 | △ 有限信息 |
| 完全无法获取 | ✗ 失败 |
